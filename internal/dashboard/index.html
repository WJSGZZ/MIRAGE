What main clients do that we still need

MIRAGE already handles local HTTP and SOCKS, but full software coverage still depends on which proxy channel each app honors.

System Proxy: good for browsers and WinINet-style apps
WinHTTP: needed by some services and app backends
Environment Proxy: catches command-line tools after restart
Full Capture: TUN / service mode is still the missing last mile

Roadmap

The next steps needed to make MIRAGE feel closer to Clash-style full desktop clients.

1Stabilize system proxy coverage

Keep WinINet, WinHTTP, and environment channels synchronized and clearly visible.

2Add true full-capture mode

TUN or service mode is the missing piece for apps that ignore every higher-level proxy channel.

3Expand profile management

Edit, reorder, duplicate, tag, and eventually sync profiles like mainstream clients do.

4Desktop-native behaviors

Tray, single instance, startup options, and richer logs should sit beside the Go core, not inside a browser tab.
